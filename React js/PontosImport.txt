1º React js não é um framework e sim uma biblioteca baseada em 
componentes.
2º Tudo em React é componente, tudo que é criado utilizando React js vai ser componente

O que são componentes em Javascript?
São funções ou classes, sempre que tiver se tratantando em uma lib(biblioteca) baseada em componentes
vai se tratar de uma função ou classe.

* React trabalha com unico sentido chamado de  One-way data flow ou One-way binding.
Para passar informção para os componentes é de cima para baixo.

Desvantagem
Que pode ficar muito custoso, imagina que tivesse um projeto grande onde há dezenas ou centenas de componentes(Pais, filhos, irmãos)
tendo a tafera de propagar informações de cima para baixou ou de baixo pra cima.
Seria inviavel.

Como Resolver: Pra isso existem bibliotecas para resolver esse problema
a lib mais famosa é CONTEXT API
